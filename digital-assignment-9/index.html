The Curator

  • Originiality:
    • The idea behind the game is original as it takes place in a library in which you have to organize a book shelf full of books. Although it is a simple puzzle game, it can be expanded on with additional functionality that can make the game more challenging. Furthemore, all of the aspects of the game and the game objects that are present are all original as I coded them all in using Phaser.
    • Iteration: The originiality of the game is the same for the iteration.
  • Technical Merit:
    • The technical merit of this game ties into the core mechanic which is moving the books into different locations in order to put them in the correct alphabetical order. The game precisely moves books from different stacks into other stacks or specific loactions. The game also has a timer implemented that adds an additional level of challenge to the game. The mouse controls all of the input in the game because there are twelve buttons that control what happens in the game. I also had specific locations on the screen be used for the bookshelf, the stack of books, and the hands. So, when a button is clicked, a book is moved to one of those locations and this makes the locations of the books dynamic.
    • Iteration: Although the mechanic of the game at its core is relatively simple, creating and coding the technical aspects of this game proved to be relatively complex. I added in randomization which randomizes the letters on each of the books. This was a techical challenge because in order to randomize the letters, I first needed to find the UTF-8 code for all upper-case and lower-case letters. I then used these codes to generate a string after randomizing the numerical code for each of the two letters on the book. This, however, created another technical challenge in that the correct answer needed to be stored even though the answer would change for each game. I had to devise an algorithm that would iterate through an array of all of the books and compare their text members to find the book with the letters earliest in the alphabet to add onto another array that would store the final answer. I also had some trouble with callback functions and scenes in Phaser. As a result, I added functionality that checked the timer and displayed text on the screen if the player won or if they lost. I also added a second shelf and each shelf is limited to what books you can put on it (in regards to the alphabet), and I added buttons that could interact with the second shelf.
  • Prototype Postmortem:
    • I was able to learn more about game objects in Phaser since this game doesn't actually use any assets and only uses game objects such as text game objects and rectangle objects in Phaser. This was also the first game in which I had to deal with arrays in JavaScript because the arrays acted as the stacks behind the scenes which manage which books are stored where on the screen. There is also a timer, but I was unable to figure out how to make the timer appear in just minutes and seconds, but that is something I hope to expand on in the next iteration. Also, for this first iteration, I made the game deterministic, but I hope to make the letters on the books random for the next iteration. The easiest part of making the game was getting all of the game objects in the correct places on the screen. Since I have dealt with the Phaser game objects before, I knew how to place them on the screen, I just had to figure out precisely where I wanted to put them. The hardest part of making the game was learning about arrays in JavaScript for the back-end stack storage. It was also difficult implement the logic behind what happens when a button is pressed because I needed to determine what arrays were altered as well as where the book would be moved to.
    • Iteration: I learned more about how to add in random elements to Phaser games and how to connect those random elements between numerical elements and string-like elements. I encountered some trouble when attempting to add scenes to my prototype. The scene switching that I needed to implement was factored into callback functions that were called after timers finished. The additional challenges that I wanted to add into the game were based off of an event happening at certain time intervals, so I was unable to add some of these challenges in. Furthermore, I was able to get some of the callback functions to be called, but I could not add in new elements, and if I created new elements outside the callback function, the placement of those game objects were unusual and incorrect. The hardest part of making the game was getting the randomly generated letters on the books to be randomly generated correctly and for the correct answer to be stored and found correctly. The easiest part of the iteration was adding in another bookshelf to be considered.

  • Gameplay:
    • The gameplay begins with a stack of books being displayed, some books on a shelf, and the player having two hands (and twelve buttons) to work with. The goal is to align all of the books in the correct alphabetical order. The buttons can be pressed to control where the books are being stored, and these buttons are what are used to correctly align the books. If the player aligns the books correctly before time is over, they win. However, if time runs out before the books are correctly aligned, then the player loses. I hope to enhance the gameplay in the next iteration by adding in a "prankster" who takes books away and adds them in. I also hope to add in randomization of the letters on the books. These will add additional challenges in the gameplay.
    • Iteration: The gameplay is largely the same as the previous iteration with some changes. The player's goal is still to sort the books in alphabetical order. However, there is an additional bookshelf. Books ranging from 'A' to 'M' can be placed on the upper shelf while books ranging from 'M' to 'Z' can be placed on the lower shelf. There are eight new buttons as well that allow the player's "hands" to interact with the lower shelf. I was unable to implement the "prankster" idea that I had hoped to. I was able to implement randomization for the letters on the books each time the game is played. This adds a level of replayability.
  • Strategies:
    • I expect that the players will use the right side of the shelf as another "stack" and place books there in order to find the next book in the correct alphabetical sequence. This will allow players to quickly filter through all of the books until the next book is found.
    • Iteration: I expect that players will first try to get each book on the correct shelf, but not in the correct order. Then, I expect players to use the stack, and one side of the shelf as intermediate storage while they line up the books on the left side of the shelf.
  • Story/Setting/Premise:
    • The game takes place in one of the most famous libraries in the world. The player is a curator for this library and is tasked with quickly organizing a bookshelf with a stack of books. The player must organize the books before the head curator comes to inspect the shelf and time runs out! There are two rectangles on the screen that represent the player's "hands" which can hold books and place them on the shelf or the stack. However, the dramatic tension is getting caught by the head curator when time runs out. I hope to implement a scene in which the head curator makes an appearance in the next iteration.
